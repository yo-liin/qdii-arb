# IB 夜盘卖空下单调试总结

> 编写日期：2026-07-02
> 核心目的：记录 IB API 夜盘下单的常见坑和修复方案，防止后续 AI 反复踩坑

---

## 一、调试背景

在 ArbDashboard 实时沙盘页面通过 `ib_reader.place_us_order()` 对 GLD 卖空，LMT 375.55，使用 OVERNIGHT 交易所路由。

最终结果：✅ **下单成功，入案成交。**

## 二、错误清单（按时间顺序）

### 错误 1：Error 10268 —— 不支持"EtradeOnly"委托单属性

**现象：**
```
[4;2;2;10268;不支持"EtradeOnly"委托单属性。]
```

**原因：** `ibapi` 的 `Order` 类包含 `eTradeOnly` 和 `firmQuoteOnly` 属性。在某些 TWS/Gateway 版本中，这两个属性默认非 `False`，导致拒单。

**修复：** 在 `place_us_order()` 中显式设置为 `False`：
```python
order.eTradeOnly = False
order.firmQuoteOnly = False
```

**位置：** `arbcore/fetchers/ib_reader.py`，`place_us_order()` 方法中设置完 `outsideRth` 之后。

---

### 错误 2：orderId = 0 导致订单被拒

**现象：** TWS 日志显示 `orderId=0`，ETradeOnly 拒单后出现 orderId=0 或异常订单 ID。

**原因：** `reconnect()` 和 `connect_to_ib()` 没有重置 `next_order_id = None`，导致重连后 `reqIds(-1)` 的 `nextValidId` 回调可能不触发，订单 ID 为 None 或 0。

**修复：**
- `reconnect()` 开头加 `self.next_order_id = None`
- `connect_to_ib()` 开头加 `self.next_order_id = None`

**位置：** `arbcore/fetchers/ib_reader.py`

---

### 错误 3：连接成功但行情不推送（"等待数据"）

**现象：** IB 按钮绿灯，但前端盘口显示"等待数据"。

**原因：** `start_polling()` 只在第一次 `get_price()` 调用时懒加载触发（`market_data_service.py` 第 144-145 行）。如果前端请求没触发 get_price，或者 get_price 返回 None 就去兜底富途了，轮询线程不会启动。

**修复：** 在 `reconnect()` 连接成功后显式调用 `self.start_polling()`：
```python
if self.connect_to_ib():
    self.start_polling()
```

**位置：** `arbcore/fetchers/ib_reader.py`，`reconnect()` 方法。

---

### 错误 4：日志刷屏 —— "⏳ IB正在获取XXX，请稍后..."

**现象：** 多条日志反复出现：
```
⏳ IB正在获取GLD，请稍后...
⏳ IB正在获取USO，请稍后...
```

**原因：** 每次前端轮询调用 `get_price()` 时，只要 `ib_reader.prices` 中还没有该 symbol，就会打一次日志。87 只基金 × 15 秒轮询 = 大量刷屏。

**修复：** 添加限频机制，每个 symbol 每 30 秒只打一次日志：
```python
now = time.time()
if not hasattr(self, '_ib_wait_log_time'):
    self._ib_wait_log_time = {}
last_log = self._ib_wait_log_time.get(symbol, 0)
if now - last_log > 30:
    logger.info(f"⏳ IB正在获取{symbol}，请稍后...")
    self._ib_wait_log_time[symbol] = now
```

**位置：** `ArbDashboard/backend/services/market_data_service.py`，`get_price()` 方法。

## 三、下单流程备忘

### 3.1 夜盘交易所路由

```python
if self.is_us_night_session():
    contract.exchange = "OVERNIGHT"
else:
    contract.exchange = "SMART"
```

**夜盘判断逻辑（北京时间）：**
- 夏令时（3-11月）：08:00 - 15:50
- 冬令时（11-3月）：09:00 - 16:50
- 仅周一到周五

### 3.2 Primary Exchange 规则

核心标的（QQQ, SPY, GLD, USO, XOP, XBI, SLV）：
- **夜盘（OVERNIGHT）**：必须**移除** `primaryExchange`，否则 Gateway 的 Sec-def 断连时容易导致 Error 201 废单
- **非夜盘（SMART）**：需要设置 `primaryExchange`

```python
primary_map = {"QQQ": "NASDAQ", "SPY": "ARCA", "GLD": "ARCA", ...}
if symbol in primary_map and not self.is_us_night_session():
    contract.primaryExchange = primary_map[symbol]
```

### 3.3 卖空指令

```python
if action == "SELL":
    order.shortSaleSlot = 1
```

注意：XOP 在部分时段不可卖空（Error 201: "该合约不可卖空"），这是 IB 侧限制，不是代码问题。

### 3.4 警告 2109（良性）

```
委托单事件警告：根据委托单类型和目的地，属性"常规交易时间以外"被忽略。现在正在处理PlaceOrder。
```

OVERNIGHT 交易所本身支持盘外交易，`outsideRth=True` 标记是多余的，但无害。**这个警告可以忽略。**

## 四、TWS 日志阅读指南

TWS 日志位于 TWS/Gateway 的 `ibgateway/jts/logs/` 目录下（而非项目目录）。

关键消息格式：

| 方向 | 格式 | 含义 |
|------|------|------|
| `<-` | `[3;2;0;XOP;...]` | OUT: 下单指令。3=版本, 2=消息类型, 0=orderId |
| `->` | `[4;2;2;10268;...]` | IN: 错误响应。4=版本, 2=消息类型, 2=reqId/orderId, 10268=错误码 |
| `->` | `[1;6;reqId;tickType;price;size;...]` | IN: 实时行情推送。1=tickPrice, 2=tickSize |
| `->` | `[17;reqId;...]` | IN: 历史数据响应 |
| `<-` | `[20;reqId;...]` | OUT: 历史数据请求 |

**常见错误码对照：**

| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 10268 | EtradeOnly 属性不支持 | 设置 `order.eTradeOnly = False` |
| 2157 | Sec-def 数据场断连 | 忽略，不影响行情和下单 |
| 2104/2106 | 数据场连接正常 | 正常信息 |
| 2107 | 历史数据场未激活 | 正常信息 |
| 2109 | OutsideRth 被忽略 | 良性警告 |
| 201 | 合约不可卖空 | IB 侧限制，换标的 |
| 502/1100 | 连接断开 | 触发重连 |

## 五、AI 修改红线（永久锁定）

以下是对 `ib_reader.py` 的**永久规则**：

1. **`order.eTradeOnly = False` 不允许删除或注释掉** — 删掉 = Error 10268 拒单
2. **`next_order_id = None` 必须在 `reconnect()` 和 `connect_to_ib()` 开头重置** — 否则 orderId=0 废单
3. **`start_polling()` 必须在 `reconnect()` 成功连接后调用** — 否则行情不推送
4. **`exchange="OVERNIGHT"` 夜盘路由不得擅自修改为 SMART** — 参考测试脚本已验证
5. **`primaryExchange` 在夜盘必须留空** — 否则 Sec-def 断连时废单
6. **不允许用 `primary_map` 以外的符号覆盖逻辑** — 新增核心标的需确认 IB 支持
