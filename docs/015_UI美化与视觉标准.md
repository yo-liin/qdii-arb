# 🎨 UI/UX 视觉优化设计标准 (工业级 V2.2)

本系统在 2026 年 6 月完成了一次全面的视觉升级，旨在对标专业量化终端，提供极低视觉疲劳、高辨识度的数据看板环境。

---

## 一、 配色方案 (Color Palette)

系统采用了“柔和底色 + 高对比度指标”的色彩策略：

| 元素名称 | 色号 (Hex) | 用途说明 |
| :--- | :--- | :--- |
| **主标题字体** | `#1f2937` | 沉稳深灰蓝，用于页头标题，增加质感。 |
| **表头底色** | `#eef2f6` | 极浅蓝灰色，清爽、现代，降低大面积表格的压迫感。 |
| **表头字体** | `#1f2937` | 与标题一致，采用加粗处理，提升列名识别度。 |
| **上涨 (Bull)** | `#ef4444` | 亮红色，仅用于百分比（涨幅、溢价率）> 0 时。 |
| **下跌 (Bear)** | `#22c55e` | 亮绿色，仅用于百分比（涨幅、溢价率）< 0 时。 |
| **中性数字** | `#334155` | 深蓝灰色，用于现价、份额、代码等绝对数值。 |

## 二、 字体规范 (Typography)

系统实现了“中文柔和、英文严谨”的字体分离架构：

1.  **中文/通用字体 (`--font-sans`)**：
    *   优先：`Microsoft YaHei` (微软雅黑), `PingFang SC` (苹方)。
    *   特性：字间距 (`letter-spacing`) 设定为 `2px` (表头) 和 `8px` (主标题)，增加呼吸感。
2.  **数值/代码字体 (`--font-num`)**：
    *   优先：`Consolas`, `Monaco`, `Courier New` (等宽字体)。
    *   应用：所有数字列强制应用，确保每一位数字完美对齐，方便上下扫描对比。

---

## 三、 核心交互标准

1.  **红绿着色隔离**：
    *   **原则**：严禁对现价、点位、份额等非方向性数字进行红绿着色。
    *   **范围**：仅 `涨幅`、`溢价率`、`份额涨幅` 三列支持动态色值。
2.  **排序小箭头**：
    *   放弃笨重的黑色大三角 `▼`。
    *   采用精致的 `▾` (向下) 和 `▴` (向上)，默认颜色为 `#94a3b8`，激活状态为 `#3b82f6`。
3.  **数据净化 (NaN 策略)**：
    *   所有 `NaN` 值在渲染前必须通过 `_clean_nans` 函数强制转换为字符串 `'-'`。
    *   前端模板必须使用 `is number` 过滤器拦截 `'-'` 字符串，防止比较逻辑报错。
