# 整体架构设计说明 (Industrial Grade V2.2)

## 1. 系统愿景

LOF 基金套利监控系统旨在为专业投资者提供一个**端到端**、**高可靠**且**极低延迟**的套利决策支持环境。通过“云地协同”数据采集与“多源瀑布流”行情引擎，确保在瞬息万变的套利窗口期提供最精准的溢价参考。

---

## 2. 核心架构逻辑：工业级“铁三角”

系统遵循 **BaseApp** 标准化设计规范，将整个套利生命周期拆分为四个核心阶段，形成稳固的流水线。

### 2.1 进食 (Data Siphon) - LOF011
- **职责**：全自动数据采集与历史追溯。
- **基座**：基于 `BaseApp`，统一配置加载与多线程追溯。
- **逻辑**：优先从云端 VPS 拉取高质量同步数据，若云端缺失则启动本地 API 降级采集。

### 2.2 思考 (Calculators) - LOF012
- **职责**：纯本地静态估值精算引擎。
- **基座**：利用 `arbcore.calculators` 算法库。
- **特征**：零网络依赖，基于数据库 `fund_daily_factors` 进行毫秒级批量推演，产出 T-1 日“真值”。

### 2.3 跳动 (Real-time Hub) - LOF02
- **职责**：极速实时行情网关与 API 中心。
- **基座**：标准化 `BaseApp` 启动，内建 IB、QMT、通达信长连接池。
- **特征**：提供 WebSocket 实时推送，支持 A 股盘口五档行情与美股夜盘快照。

### 2.4 颜值 (Renderer) - LOF03
- **职责**：高性能监控面板生成。
- **特征**：采用 JS 侧沙盘推演技术，将复杂的计算逻辑前置到浏览器，实现服务器 0 压力的流畅监控。

## 4. 关键技术特性

### 4.1 模块化数据库管理器 (Composition Over Inheritance)
重构后的 `DatabaseManager` 不再是臃肿的单体类，而是通过以下组合实现职责分离：
- **FundManager**：基金净值、溢价、篮子权重。
- **MarketManager**：汇率、指数、ETF 历史价。
- **SystemManager**：同步状态、健康监测、数据湖备份。

### 4.2 统一基座 (BaseApp)
所有核心脚本均继承自 `BaseApp`，获得以下开箱即用的能力：
- **NO_PROXY**：自动屏蔽系统代理，确保国内 API 请求不因梯子挂掉或被反爬。
- **Logger**：统一格式的工业级日志，支持等级动态调整。
- **DB Connector**：自动定位项目根目录下的 `arb_master.db`。

### 4.3 交易接口防坍塌设计
`trade_manager.py` 采用延迟加载与异常熔断技术。即使某个交易终端（如 QMT）意外关闭，主服务仍能正常提供行情监控，仅在尝试下单时进行静默拦截并报警。

---

## 5. 数据一致性保障 (Consistency)

系统通过 `fund_data` 超级宽表实现“动静结合”：
- **T日盘中**：实时现价（Price）通过 `Upsert` 机制覆写。
- **T日盘后**：官方净值（NAV）由 `LOF011` 填补。
- **校准逻辑**：`LOF012` 自动运行数据探针，如果发现汇率或基准点未对齐，将拒绝计算估值并红色报警，从源头杜绝虚假溢价。
