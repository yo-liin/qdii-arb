## 🌟 1. 系统核心理念：代码与数据的“动态解耦”

本项目遵循现代化量化系统的标准结构：
- **零预置数据**：你下载的代码不含任何历史数据库文件，确保了代码的轻量化与隐私安全。
- **自动初始化**：系统具备“魔法建库”能力。首次运行时，它会自动检测并创建工业级数据库，并从互联网实时抓取最新的因子数据填充进去。

## 🛠️ 2. 首次启动：见证“工业级”建库

当你准备好 Python 3.11+ 环境后，按照以下步骤操作：

### 操作步骤：
1. 进入 `LOFarb` 文件夹。
2. 双击运行 **`LOF_start_lof_system.bat`**。

### 后台正在发生的“黑科技”：
1. **清理环境**：自动杀掉卡死的旧进程，释放 5000 端口，确保启动成功。
2. **自动化建库**：底层 `DatabaseManager` 发现缺失数据库时，会立即在 `arbcore/database/` 下创建 **`arb_master.db`** (WAL 高并发模式)。
3. **数据感官启动 (LOF011)**：基于 `BaseApp` 的更新器会化身爬虫，从外汇局、Woody API 等源头抓取最新的中间价、ETF 价格及校准因子。
4. **探针自检 (LOF012)**：精算引擎会运行 **“深度诊断探针”**，通过 ✅/❌ 图形化展示你的本地数据是否与官方对齐。
5. **实时网关 (LOF02)**：启动行情网关（5000 端口），建立 IB/QMT/通达信长连接。

## 🔐 3. 配置你的交易账号

**账号信息属于最高隐私，严禁上传 Git。**

## 🔍 4. 常见问题：为什么提示“缺失数据”？

因为你的数据库是“全新出厂”的，计算 T-1 日估值需要历史基准点。

**探针报警示例**：
```text
  🕵️ [数据探针] 易方达纳100(161130) 最近3个净值日的数据对齐情况:
    ⚠️ 2026-06-02 (NAV=4.188): 缺失 -> 人民币中间价
```
**解决方案**：
- **云端追溯**：运行 `LOF011` 时，系统会尝试从远程 VPS 拉取最近 30 天的历史快照，通常能自动解决此问题。
- **手动补全**：若云端也缺失，请访问配置中心 (5001端口) 手动录入或使用历史数据导入脚本。
